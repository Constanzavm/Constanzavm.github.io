Sprint - Base de datos

Modulo 3: my SQL

Objetivo:

Crear, poblar y realizar consultas en una base de datos my SQL

Actividad:

Crear una base de datos adecuada para una tienda online, incluyendo tablas tales como usuarios/clientes, pedidos, productos, etc. Y responder una serie de preguntas relacionadas.

Proceso:

En grupo creamos y poblamos la base de datos en mySQL workbench, guardando los pasos en un script al que luego añadimos consultas para responder a las siguientes preguntas:

  • Cuál es la categoría de productos que más se repite.
  • Cuáles son los productos con mayor stock
  • Qué color de producto es más común en nuestra tienda.
  • Cual o cuales son los proveedores con menor stock de productos.

Accidentados

Modulo 6: Java Web Spring-boot

Objetivo:

Crear una página web dinámica utilizando Spring boot

Actividad:

En 2022, basados en un caso ficticio, los índices de accidentabilidad, en los distintos rubros industriales, han aumentado, generando una excesiva cantidad de información que sobrepasó a los sistemas tradicionales. Debido a esto se nos solicitó crear una aplicación capaz de responder a las necesidades de una empresa que requería estas funcionalidades. Esta aplicación fue diseñada con el objetivo de implementar un sistema de datos que permita administrar la información que genera una compañía, en este caso de capacitaciones. Para lograr esto, se utilizaron distintas herramientas, entre las que destacamos html, javascript, bases de datos relacionales en lenguaje SQL y SpringBoot.

Proceso:

En éste trabajo se realizó aquello que fue solicitado y descrito en la introducción de manera que, creándose distintos perfiles a través del uso de Spring Security, se logró discriminar entre usuarios y acceder a las funcionalidades asignadas a cada uno de ellos. Es así que un ente de tipo “Administrativo” tiene la capacidad realizar, de modo exclusivo, las funciones creadas de:

  1. Listar usuarios.
  2. Crear Usuarios.
  3. Editar Cliente.
  4. Editar Administrativo.
  5. Editar Profesional.
  6. Listar Visitas.
  7. Listar Pagos.
  8. Crear Pagos.
  9. Administrar Chequeos.
Mientras que aquellos usuarios de tipo “Cliente” tienen la facultad exclusiva de acceder a las funciones de:
  1. Contacto.
  2. Crear Capacitación.
  3. Listar Capacitación.
  4. Administrar Asistentes.
El tercer tipo de usuario, los de tipo “Profesional”, pueden acceder de manera exclusiva a funcionalidades tales como:
  1. Listar Asesorías.
  2. Responder Checklist.
  3. Listar Visitas.
  4. Crear Asesorías.
Junto con lo anterior, el software desarrollado tiene la facultad de darle acceso a los distintos perfiles acceso a la página de inicio posterior al proceso de Login.
